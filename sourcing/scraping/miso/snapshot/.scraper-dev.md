# MISO Snapshot Scraper Configuration

## Data Source Details

**Organization**: MISO (Midcontinent Independent System Operator)
**API Type**: Public REST API
**Authentication**: None required
**Documentation**: /tmp/miso-pricing-specs/spec-public-snapshot.md

## Collection Strategy

This scraper collects real-time grid status snapshots from MISO's public API. Each execution fetches the current state of 4 key grid metrics:

1. Forecasted Peak Demand (MW)
2. Current Demand (MW)
3. Marginal Energy Cost ($/MWh)
4. Scheduled Interchange (MW)

## Scheduling Recommendations

- **Production**: Run every 5 minutes
- **Development**: Run every 15 minutes
- **Testing**: Manual execution as needed

## Response Structure

```json
[
  {
    "t": "Metric Title",
    "v": "Numeric Value",
    "d": "ISO 8601 Timestamp",
    "id": "metric_identifier"
  }
]
```

## Validation Rules

1. Response must be an array with exactly 4 elements
2. Each element must contain: t, v, d, id
3. All metric IDs must match expected set
4. All values (v) must be numeric

## Storage

- **S3 Path**: `s3://{bucket}/sourcing/miso_snapshot/{date}/snapshot_{timestamp}.json.gz`
- **Compression**: gzip
- **Deduplication**: Redis hash-based
- **Retention**: Indefinite (subject to S3 lifecycle policies)

## Monitoring

- Alert if no new data for 30 minutes
- Alert if validation failure rate exceeds 5%
- Monitor duplicate rate (normal: <5%)
- Track API response times (normal: <5 seconds)
